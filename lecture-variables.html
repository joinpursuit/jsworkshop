Lecture: Variables

What is a Variable?

A good analogy is a labeled box. Outside is the label, inside is something, and it could be anything!

(Stolen from Khan Academy)

Officially: A variable is a container. It has both a name and a value. You declare a new variable with the var keyword, and assign it (aka give it) a value with the = sign. 

var variableOne = 1
var variableTwo = "Two"
var bensFavLanguage = "JavaScript"
console.log("Ben's favorite language is: " + bensFavLanguage)
// Ben's favorite language is: JavaScript

In JavaScript the value of a variable can change, but the name cannot. However, you can assign variables with values from other variables. 

var bensFavLanguage = "JavaScript"
var studentsFavLanguage = "French"
console.log("Student's favorite language was " + studentsFavLanguage)
// Student's favorite language was French

// .... after this lecture ....
studentsFavLanguage = bensFavLanguage
console.log("Student's favorite language was " + studentsFavLanguage)
// Student's favorite language was JavaScript

Notice how in the code snippet above our second console.log statement is identical to the first console.log statement but the output is drastically different since the value of the var studentsFavLanguage has changed. In general, after having declared a new variable, it can be used and reused as many times as you want.

As usual, the w3schools page on variables is a fantastic resource worth reading and visiting often.

Variable Types

What's the difference between: 

var label = "Do Not Disturb";
// and
var distance = 100;

The short answer is this: the first, var label is a string whereas the second, var distance is a number. In JavaScript, variables may have different types and each type of variable is used to represent a specific group of data.

Numbers

Numbers are what you would expect them to be in the real world — numerical values that can be added, subtracted, multiplied, divided, etc. 

// Examples
var radius = 5; 
var g = 9.81; // decimals count as numbers
var piAsFraction = 22 / 7; // this is computed and stored as 3.142857142857143

We use the arithmetic (+,-,*,/) operators on numbers to do useful things like calculate someone's age or convert temperatures from fahrenheit to celsius.

Let's look at some examples. 

// Examples

// calculating the circumference of a circle
var radius = 5; 
var PI = 3.14;

// QUESTION: does the value of `PI` or `r` change after this operation?
console.log( 2 * PI * r );

// converting F -> C
var F = 82;
// notice how parenthesis can be used for order of operations
var C = ( F - 32 ) / 1.8; 

// QUESTION: what is stored inside the `C` variable?
console.log( C );

(Optional) Challenge: can you write an expression using variables that will take our C value and convert it back to the fahrenheit scale?

Strings

A string can be used to represent words and sentences. Anything wrapped in quotations (either double or single) will be represented as a string in javascript (even if the thing in quotes is a number).

Most arithmetic operators (-,*,/) do not work with strings. The + operator can be used to "add" or concatenate strings together.

Let's see this in action. 


var name = "John"; // double quotes, this is string
var lastName = 'Smith'; // SINGLE quotes, still string
var fullName = name + " " + lastName; // John Smith
// notice how the '+' in this case combines three strings into one string

var stillAString = "2"; // this is still a string since wrapped in quotes

var invalid = "John Smith'; // NOTE: the quote TYPES mismatch, not kosher

Good Variable Names

Having good variable names is super important for you and others reading your code. You want the casing to be consistent across all the files in your project, and ideally consistent with the standards of the language. I like JavaScript Standard Style but not everyone has the same style. The most popular casing for variables in JavaScript is called "Camel Cased" which starts with a lower case letter and only capitalizes the first letter of each internal word. Examples: myNumber, sortedPrimeNumbers, listOfBestPizzaInNyc.

You want your names to be as concise and as clear as possible. Being able to reason about what data is in a variable in the middle of a big and complicated program is important both for yourself and for when you share code. You will read more code than you will write as a programmer.

Best Practices

Variable names can contain:

  • letters
  • numbers
  • underscores
  • dollar signs

Variable names cannot contain:

  • spaces
  • dashes
  • periods

Your variable names cannot start with a number, however you may choose to start your variable names with letters (lowercased or uppercased based on your discretion), underscores, or dollar signs.

Here are some examples of bad and better variable names.

// BAD variable names
var 3.14isPi // can't start with number, can't have periods
var hello everyone! // space and '!' not allowed
var this-is-also-not-kosher-as-a-variable-name-for-use-later
// has dashes, OPINION: too long, concise names are better

// BETTER variable names
var pi // all lower cased, simple and succinct 
var helloEveryone // 'camel cased': individual words are capitalized
var listOfAllUniqueNames // describes the type of data inside: a list
var orderedListOfPrimeNumbers // describes the type and the organization: a sorted list

Evaluation / Execution of code

The word we use to describe turning a set of code into a single value is called evaluate or execute. This is useful to think about in situations where we assign the result of some math to a variable: 

var x = 10
var y = 20
var someVariable = (100 * x / y) - x*y
x = 0
y = 0
console.log(someVariable)
// -150

The value of some variable is decided at the moment of assignment when that line of code gets executed and the expression on the right side of the assignment gets evaluated.

Useful and important syntax

var someNumber = 0
var x = 5 // We'll use this later

//Increment: take a value and increase it by 1
someNumber = someNumber + 1
someNumber += 1 // This and the above line can be any number
someNumber++
++someNumber

//Decrement: take a value and decrease it by 1
someNumber = someNumber - 1
someNumber -= 1 // This and the above line can be any number
someNumber--
--someNumber

//Multiply: take a value and multiply it by x
someNumber = someNumber * x
someNumber *= x

//Divide: take a value and divide it by x
someNumber = someNumber / x
someNumber /= x
